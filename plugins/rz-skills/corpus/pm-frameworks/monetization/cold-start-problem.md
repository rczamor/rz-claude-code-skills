# Cold Start Problem

## What it is
Andrew Chen's framework from *The Cold Start Problem* (2021), drawing on his work at Andreessen Horowitz and Uber. The cold-start problem is the chicken-and-egg challenge facing any network-effects product: nobody wants to use it because nobody else uses it. Chen's stages: **Cold Start** → **Tipping Point** → **Escape Velocity** → **Hitting the Ceiling** → **The Moat**. The early-stage tactics: build for an "atomic network" (the smallest functional network — Tinder's first college, Uber's first city), use single-player utility before network kicks in, subsidize the harder side of two-sided markets, piggyback on existing networks.

## Why it matters
Network-effects products dominate modern winners (Uber, Airbnb, Slack, Discord, OpenSea, etc.) but the cold-start phase kills most of them. The product is unusable without the network; the network won't form without the product. Chen's framework names the specific tactics that work pre-network — converting the cold-start phase from a vibe to a playbook. For PMs at Helm Labs–scale early stage, this is the most important monetization framework, even before pricing matters.

## How to apply
1. **Define the atomic network.** What is the smallest possible network where the product is genuinely valuable? Tinder: ~500 users on one campus. Uber: drivers + riders in one neighborhood.
2. **Build single-player utility.** What value can a user get without the network? Notion: personal notes. Twitter early: status updates without followers. Pure-network products without single-player utility almost always fail cold start.
3. **Subsidize the hard side.** In two-sided markets, identify the harder side (usually supply) and over-invest. Uber paid drivers; OpenTable comped restaurants; Airbnb staged photography for hosts.
4. **Piggyback existing networks.** Hotmail piggybacked email contacts; PayPal piggybacked eBay; Instagram piggybacked iPhone cameras + Twitter sharing.
5. **Spread by city, by college, by category — not horizontally.** Atomic-network thinking means winning one cluster fully before moving to the next.

## Examples
1. **Tinder's college-by-college rollout.** Started at USC sororities. Atomic network = one college. Once one college had density, the next was easier. Today: cold-start playbook for dating apps.
2. **Uber's neighborhood-by-neighborhood rollout in San Francisco.** Drivers and riders in one area, then expanded outward. Rather than launching nationally, atomic-network density first.
3. **Riché's Helm Labs $3.25M pre-launch pipeline** as a cold-start solution — the pipeline was built by piggybacking existing relationships and demonstrating value to one buyer cluster before broadening.

## Anti-patterns
1. **Horizontal launch.** Trying to launch nationally or globally before any atomic network has density. Spreads thin, builds no network anywhere.
2. **Pure network play with no single-player utility.** Product is useless until the network forms; users churn before the network forms. Always provide value at N=1.
