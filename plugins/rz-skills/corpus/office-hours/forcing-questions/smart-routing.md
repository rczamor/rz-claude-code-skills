# Smart Routing — Which Questions for Which Stage

## What it is
A routing table that maps the founder's product stage to the subset of the Six Forcing Questions that should be asked. You don't always need all six — only the ones whose answers aren't yet clear from the conversation.

## Why it matters
Asking Q1 (demand reality) of a founder with paying customers wastes time and signals that you weren't listening. Asking Q6 (future-fit) of a pre-product founder is premature — the future doesn't matter if no one wants this *now*. Routing matches the question to the actual diagnostic gap.

## How to apply

**Smart routing based on product stage — you don't always need all six:**

| Stage | Required questions |
|---|---|
| Pre-product | Q1, Q2, Q3 |
| Has users | Q2, Q4, Q5 |
| Has paying customers | Q4, Q5, Q6 |
| Pure engineering / infra | Q2, Q4 only |

**Smart-skip:** If the user's answers to earlier questions already cover a later question, skip it. Only ask questions whose answers aren't yet clear.

**STOP after each question.** Wait for the response before asking the next.

**Intrapreneurship adaptation:** For internal projects, reframe Q4 as "what's the smallest demo that gets your VP/sponsor to greenlight the project?" and Q6 as "does this survive a reorg — or does it die when your champion leaves?"

## Examples

1. **Pre-product founder, no users yet:** Run Q1 (demand reality) → Q2 (status quo) → Q3 (named human). The diagnostic question is "does anyone want this at all?"
2. **Founder with 50 users, no revenue:** Run Q2 (workaround cost) → Q4 (wedge) → Q5 (observation surprise). The diagnostic question is "is the wedge sharp enough to monetize?"
3. **Founder with $50K MRR:** Run Q4 (wedge) → Q5 (surprise) → Q6 (future-fit). The diagnostic question is "is this fundable, or commodity?"
4. **Engineering-only project:** Run Q2 (workaround cost) → Q4 (smallest paid version). User research and future-fit aren't where this project will fail.

## Anti-patterns
- Running all six questions out of completionism. The questions are diagnostic tools, not a script to finish.
- Skipping all of them because the founder seems prepared. Even fully-formed plans still get Phase 3 (Premise Challenge) and Phase 4 (Alternatives).
- Asking Q1 of a paying-customer founder. They've passed the demand-reality test by definition.
