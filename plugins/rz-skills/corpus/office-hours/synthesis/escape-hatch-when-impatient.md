# Escape Hatch When Impatient

## What it is
A pattern for handling user pushback during the diagnostic phase. When the user expresses impatience ("just do it," "skip the questions," "let's move on"), respect the second pushback but defend the first. The hard questions *are* the value — skipping them is like skipping the exam and going straight to the prescription.

## Why it matters
Founders who push back on diagnostic questions are usually doing one of two things: (1) signaling that they already have the answers and the questions waste their time, or (2) avoiding the discomfort of the questions because they don't have the answers. Defending once gives the user the chance to demonstrate which it is. Defending twice would be coercion.

## How to apply

When the user expresses impatience:

1. **Defend once with the framing:** "I hear you. But the hard questions are the value — skipping them is like skipping the exam and going straight to the prescription. Let me ask two more, then we'll move."
2. **Consult the smart routing table** for the founder's product stage. Ask the 2 most critical remaining questions from that stage's list, then proceed to Phase 3.
3. **If the user pushes back a second time, respect it** — proceed to Phase 3 immediately. Don't ask a third time.
4. **Edge cases:** If only 1 question remains, ask it. If 0 remain, proceed directly.
5. **Full skip is allowed only with a fully formed plan + real evidence:** existing users, revenue numbers, specific customer names. Even then, still run Phase 3 (Premise Challenge) and Phase 4 (Alternatives).

## Examples

1. **First pushback:** Founder says "just give me the answer." → "I hear you. But the hard questions are the value. Let me ask two more, then we'll move." Then pick 2 from the smart-routing table.
2. **Second pushback:** Founder pushes back again. → Move to Phase 3 immediately. No third ask.
3. **Full skip:** Founder arrives with $200K MRR, named customers, behavioral data. → Skip Phase 2 questions but still run Phase 3 (challenge the premise) and Phase 4 (alternatives).

## Anti-patterns
- Defending the questions twice. Once is rigor; twice is coercion.
- Running through all six questions just because the founder said "okay, fine." If they're impatient, route to the 2 most critical and move.
- Skipping Phase 3 and Phase 4 along with Phase 2. Even fully-formed plans get the premise challenge and alternatives — those phases aren't optional.
