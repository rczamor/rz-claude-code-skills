# X Thread Format (5–7 Tweets)

## What it is
The format for adapting LinkedIn deep dives into X-native threads: 5–7 tweets, each tweet standalone-readable, leading tweet hook-driven, last tweet either a CTA or punchline. X norms reward terse, punchy, opinionated, technical — the opposite of LinkedIn's longer-paragraph format.

## Why it matters
Cross-platform copy-paste fails. A LinkedIn paragraph on X reads as wall-of-text; an X-native thread on LinkedIn reads as fragmented. The 5–7 tweet thread is the X-native unit that delivers the same substance as a Wednesday LinkedIn deep dive in a format X actually amplifies. Without the format adaptation, X presence is dead weight.

## How to apply
- **Lead tweet = hook.** Punchier than the LinkedIn opener. "Most AI PMs prioritize features when they should prioritize context surface area." Stand-alone reading.
- **Tweets 2–6 = the substance.** Each tweet a single idea. Use line breaks within a tweet for emphasis. Numbered if a list, otherwise prose.
- **Last tweet = CTA or punchline.** A demo URL, a "what's the framework you use?" provocation, or the punchline that brings the thread home.
- **Length per tweet:** aim for 200–280 chars. Don't pad. Don't cram.
- **Use threading deliberately.** Each tweet earns its own quote-tweet potential. Quote-tweetability is part of the design.
- **Cross-post timing:** Tuesday, ahead of Wednesday LinkedIn deep dive. Founders see X first; PMs see LinkedIn next day.

## Examples
Tuesday X thread (adapted from Wednesday LinkedIn deep dive on context architecture):

> 1/ Most AI PMs prioritize features. The teams shipping breakthrough AI products prioritize context surface area first.
>
> 2/ Context architecture: how the system represents what the user knows, has done, and intends. Not the model. Not the UX. The substrate underneath both.
>
> 3/ When we launched Suzy with 350+ enterprise customers, our early disengagement read like PMF failure. It was context drift. Different bug.
>
> 4/ The diagnostic: are users disengaging because the product doesn't fit their needs, or because the system can't see their context?
>
> 5/ Most AI PM job descriptions miss this layer entirely. They list "prompt engineering" or "model selection" — the surface, not the substrate.
>
> 6/ I built the Context Layer Engine to externalize this for myself. Live demo: [URL]. Five-step process. Architecture in the open.
>
> 7/ If you're an AI PM and your roadmap is mostly features, ask: what's the context surface area of this product? That's the question most teams skip.

## Anti-patterns
- Copy-pasting LinkedIn paragraphs as multi-tweet threads. Format mismatch; reads as imported.
- Threads with 12+ tweets. X norms cap attention at 5–7; longer threads lose readers mid-thread.
