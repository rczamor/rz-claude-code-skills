# Pattern 3 — Platform Vision → Wedge Challenge

## What it is
A pushback pattern for when the founder insists they need to build the full platform before anyone can really use it. The move is to name this as a red flag — usually the value prop isn't clear yet, not that the product needs to be bigger — and demand the one thing a user would pay for this week.

## Why it matters
Founders fall in love with architecture. "We need the full platform" is almost never about the user — it's about the founder's mental model of how the system fits together. The wedge challenge forces them to extract the single most valuable thing from inside the platform and ship it standalone. If they can't, the platform vision is hypothesis dressed up as product.

## How to apply

When you hear platform-first language:
1. Name the red flag: "If no one can get value from a smaller version, the value prop isn't clear yet."
2. Demand the wedge: "What's the one thing a user would pay for this week?"
3. If they push back ("we'd need to keep most of the architecture") — that's the founder defending architecture, not value.

## Examples

**Founder:** "We need to build the full platform before anyone can really use it"

**BAD:** "What would a stripped-down version look like?"

**GOOD:** "That's a red flag. If no one can get value from a smaller version, it usually means the value proposition isn't clear yet — not that the product needs to be bigger. What's the one thing a user would pay for this week?"

The BAD response accepts the platform framing and asks for a stripped-down version inside it. The GOOD response names that "needs the full platform" is itself the diagnostic signal — the value prop isn't sharp.

## Anti-patterns
- Asking "what would a stripped-down version look like" — that's the BAD response. It accepts the platform framing.
- Letting the founder define the wedge as "the MVP version of the platform." A wedge is a standalone thing someone pays for.
- Accepting "we'd need to keep most of the architecture for it to work." Translation: the founder is attached to the architecture, not the value.
